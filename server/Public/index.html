Documentação da API

Bem-vindo à documentação da nossa API. Esta página fornece todas as informações necessárias para integrar com nossos serviços.
URL Base: https://api.exemplo.com/v1

Autenticação

Todas as requisições à API devem incluir um token de autenticação no cabeçalho:

Authorization: Bearer SEU_TOKEN_AQUI

Endpoints

GET /usuarios

Retorna uma lista de todos os usuários cadastrados.

Exemplo de Requisição

GET https://api.exemplo.com/v1/usuarios

Exemplo de Resposta

{
  "data": [
    {
      "id": 1,
      "nome": "João Silva",
      "email": "joao@exemplo.com",
      "data_criacao": "2023-01-15T10:30:00Z"
    },
    {
      "id": 2,
      "nome": "Maria Souza",
      "email": "maria@exemplo.com",
      "data_criacao": "2023-02-20T14:45:00Z"
    }
  ],
  "total": 2
}
POST /usuarios

Cria um novo usuário no sistema.

Parâmetros

Nome Tipo Obrigatório Descrição
nome string sim Nome completo do usuário
email string sim E-mail do usuário
senha string sim Senha de acesso (mínimo 8 caracteres)

Exemplo de Requisição

POST https://api.exemplo.com/v1/usuarios
Content-Type: application/json

{
  "nome": "Carlos Oliveira",
  "email": "carlos@exemplo.com",
  "senha": "senhaSegura123"
}

Exemplo de Resposta

{
  "id": 3,
  "nome": "Carlos Oliveira",
  "email": "carlos@exemplo.com",
  "data_criacao": "2023-05-07T09:15:00Z"
}

Códigos de Status

Código Descrição
200 OK - Requisição bem-sucedida
201 Created - Recurso criado com sucesso
400 Bad Request - Parâmetros inválidos
401 Unauthorized - Autenticação necessária
404 Not Found - Recurso não encontrado
500 Internal Server Error - Erro no servidor

Limites e Cotas

  • Limite de requisições: 100 requisições por minuto por token
  • Tamanho máximo de payload: 1MB
  • Timeout: 30 segundos por requisição
